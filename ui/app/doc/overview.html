What is Notes?

Notes is a web based, offline capable Progressive Web App for notes taking. It is compatible to every device running a recent browser, but can also be installed locally as a PWA. All functionality is also available when there is no internet connection, and optimized for mobile as well as desktop usage. 

The idea to create this was the lack of existing major note taking apps which are either too unstable or intransparent when it goes about synchronizing errors and conflicts, too big and unresponsive, or did not provide all functionality i needed personally. For example, here you can have kanban boards inside tree structured documents, which can have further sub-boards etc, not limited in any way. Also there is both WYSIWYG and code style editing in the same app, which is also unique. Navigation is supported with in-document links and backlings, similar to Obsidian.

The app stores and manages textual notes, PDF documents, images and any other type of attachment files, in an unrestricted hierarchical way. This means that every document can have child documents, as deep as necessary. There are multiple ways of accessing and querying the documents, as well as multiple editors to choose from, providing rich text (WYSIWYG) editing as well as markdown based and source code editors. Additionally, the app provides a kanban board view similar to Trello or the like, all integrated in one place. The app can manage multiple different notebooks (each represented by a CouchDB database) and switch between them.

The documents can be labeled, referenced, and there is a history for all textual documents so you never loose anything by accidentally deleting a note or parts of it. 

There are multiple ways of importing and exporting documents, for example Trello boards can be imported directly. The app is Open Source, so you can add your own importers/exporters if you are familiar with Javascript. See the technical documentation for more details on this.

Technical Overview

The app is based on pure HTML/Javascript and a CouchDB backend. This database is document based, so it is well suitable for this application, and also the PouchDB library provides the synchronizing mechanisms to be able to fully use the app when there is no network connection to the CouchDB server.

The data is stored in JSON format (native to CouchDB) internally. One notebook is represented by one database in CouchDB. A copy of the database containing the complete notebook can be held locally in the browser's app storage, so if you make changes to a document while being offline, the changes will be synchronized whenever there is a connection to the server again (this is what PouchDB does).

If there are conflicts with other client instances changing the same document, the user is noticed about this and has to decide which version is the correct one (manual conflict resolution, described in the Usage Documentation).

Requirements

The UI part of the app (HTML and JavaScript) is hosted as a purely static web page on any simple web server which can serve HTML and JS files. You can clone the Github repository to a server of your own or simply use this link if you don't want to host it yourself or just want to try it out:

https://notes.webertom.net

The app has been tested on recent Chrome and Safari browsers, on Windows, MacOS and iOS. Others might work but in case of errors, please try one of these.

The server part is simply represented by a CouchDB instance, no PHP or any other server side scripts are needed. You can either deploy your own CouchDB instance on your own server (CouchDB is available for a variety of operating systems and environments), or use a hosted database provider. 

NOTE: For an initial test, it is also possible to just work with the local database of the app. You can test all functionality (except synchronizing with a remote database of course) of the app before you get hold of a CouchDB instance, which might cause further setup work or costs (if you choose a hosted service for example). Just open the app in your browser and choose the "local" notebook, see the Usage documentation for details.

In case you run into cross site origin (CORS) problems (the browser blocks CORS requests by default if the server is not set up correctly), you can:

  • Change the CouchDB server settings to allow cross domain access
  • Install a proxy script in the same location as you put the app sources (only possible if you host the app yourself, see chapter "Hosting" below). For example, you can use this PHP-only proxy script which redirects REST requests to another host. The URL entered in the app for opening the notebook would then be the URL of the proxy script with paramaters according to the documentation.

Hosting the App yourself

If you want to host the app yourself, just clone the repository from Github into the htdocs (or similar) folder of a web space which could be a standard Apache server for example. No PHP is required for the core app.
